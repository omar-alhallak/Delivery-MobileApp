# Delivery App Clean Architecture
تعتمد Clean Architecture على تقسيم المشروع إلى طبقات رئيسية مثل:
- Domain: يحتوي على الكيانات والقواعد الأساسية للنظام.
- Application: يحتوي على منطق التطبيق وحالات الاستخدام.
- Infrastructure: يحتوي على التعامل مع قواعد البيانات والخدمات الخارجية.
- API: الطبقة التي تتعامل مع المستخدم .
## 🌳 Branching
### Branch (`main`) : → فرع الإنتاج 
* A_ ❗ يمنع التعديل المباشر عليه.
* B_ ❗ يمنع إنشاء فروع منه.
### Branch (`Develop`) : → فرع التطوير الرئيسي 
### Branch (`feature/*`) : → فروع الميزات

## طريقة العمل على البيئة:
1️⃣ أولاً: تقوم بإنشاء فرع feature/ من فرع Develop للميزة التي ترغب بإضافتها إلى Develop.

2️⃣ تقوم بعمل Puch من فرع feature/ الذي تم إنشاؤه ليظهر بال Git.

3️⃣ تبدأ بالتعديل أو إضافة أي شيء تريده أو أي عمل تقوم بتنفيذه.

4️⃣ عند الانتهاء من العمل، تكون طريقة الحفظ كالتالي:
   أ) أقوم بعمل Commit لحفظ التغيرات، ومن ثم Puch لرفع العمل على الغيت.
5️⃣ إذا أردت أن تحدث برانشك تقوم بعمل Pull request عكسي الخطوات:
   أ) أقوم بعمل Merge لفرع Develop إلى feature/.
  ب) من فرعي feature/ أقوم بعمل Pull, ليتحدث الفرع.
6️⃣ عند الإنتهاء من العمل أقوم بعمل Pull request لفرع feature/ إلى Develop، وبذلك أكون قد انتهيت من feature/.
6️⃣ أقوم بعمل Pull لفرع Develop.

⚠️ القواعد الصارمة للعمل : 
1️⃣ في حال تم إجراء عملية Merge للـ feature/ ونسيت إضافة شيء ما، يمنع التعديل على نفس الفرع مرة أخرى منعاً باتاً.
يجب حذف الفرع وإنشاء feature/ جديدة وإجراء التعديلات عليها.
2️⃣ يمنع منعاً باتاً تعديل أو إضافة أي شيء على:
Dependencies.
Migrations.
DeliveryApp.Domain.
Branch(main).
أي تعديل عليهما قد يؤدي إلى خلل في المشروع بالكامل.
3️⃣ ممنوع منعاً باتاً إجراء Merge إلى فرع main.
فرع main محمي بالكامل ولا يسمح لأي شخص بالتعديل عليه أو الاقتراب منه.
4️⃣ يُمنع حذف أو تعديل أي سطر مباشرة على Develop أو main.
في حال نسيان أي تعديل، يجب إنشاء فرع feature/ جديد وتنفيذ التعديل من خلاله فقط.
5️⃣ يمنع رفع أي كود يحتوي على أخطاء (Errors) أو مشاكل (Bugs) إلى Develop.
يجب التأكد من أن الكود يعمل بنسبة 100% قبل رفعه و دمجه.
6️⃣يمنع منعاً باتاً إضافة أو حذف أي مكتبة.
⚠️ أي مخالفة لأي بند مما سبق قد تؤدي إلى تعطل المشروع بالكامل.

ملاحظات(مهمة):
1️⃣ كل Feature لازم تكون صغيرة وواضحة لا تجمع أكثر من مهمة بفرع واحد.
كل Feature لازم تحل مشكلة واحدة فقط أو تضيف ميزة واحدة فقط.
2️ تأكد قبل الدمج من الأمور التالية
قبل عمل Merge إلى Develop:
لا يوجد Errors
لا يوجد Warnings خطيرة
لا يوجد Console Logs
لا يوجد Debug Code
تم اختبار الـ API يدوياً
3️⃣مثال على كيفية تسمية Branch:
feature/add-user-login

في حال وجود أي استفسار أو مواجهة أي مشكلة أو عدم وضوح في أي جزء من بيئة العمل،
أو في حال الرغبة بإضافة مكتبة، أو تعديل أي جزء من المشروع من الأجزاء التي يمنع التعديل عليها —
يجب التواصل معي أولاً قبل القيام بأي إجراء.
⚠️ يُمنع اتخاذ أي قرار فردي يتعلق بتعديل أجزاء محمية أو حساسة في المشروع.
